## 4. Recomendação de Schema

### 4.1 Tipo `item`

Entidade única `item` com campo de categoria discriminante:

```typescript
type CategoriaItem =
  | "arma"
  | "armadura"
  | "escudo"
  | "municao"       // flechas, virotes, balas, pedras
  | "item-aventura"
  | "ferramenta"
  | "vestuario"
  | "esoterico"
  | "alcuimico-preparado"
  | "alcuimico-catalisador"
  | "alcuimico-veneno"
  | "alimentacao"
  | "animal"
  | "veiculo"
  | "servico"
```

### 4.2 Campos comuns (todos os itens)

```typescript
interface ItemBase {
  id: string;
  nome: string;
  categoria: CategoriaItem;
  preco: string | null;        // ex: "T$ 15", "—" (sem custo), null
  espacos: number | "meio" | null; // 0.5 para itens alquímicos, null para animais/veículos/serviços
  descricao: string;           // texto narrativo
  fonte: { livro: string; pagina: number };
}
```

### 4.3 Campos específicos por categoria

#### Arma
```typescript
interface ItemArma extends ItemBase {
  categoria: "arma";
  proficiencia: "simples" | "marcial" | "exotica" | "fogo";
  proposito: "corpo-a-corpo" | "distancia";
  subtipo_distancia?: "arremesso" | "disparo";
  empunhadura: "leve" | "uma-mao" | "duas-maos";
  dano: string;                // ex: "1d8", "2d4/2d4" (adaptável), "3d4"
  dano_duas_maos?: string;     // segundo valor de dano para armas adaptáveis (espada bastarda, katana, corrente)
  critico: string;             // ex: "19", "x3", "19/x3", "x4", "18"
  alcance?: "curto" | "medio" | "longo";
  tipo_dano: string;           // "Corte" | "Impacto" | "Perfuração" | "Corte/perfuração"
  habilidades: string[];       // ["ágil", "adaptável", "dupla", "versátil", "alongada", "desbalanceada"]
  bonus_manobra?: string;      // ex: "+2 em testes para derrubar" (derivado de habilidade Versátil)
  especial?: string;           // regras únicas da arma (ex: Rede — enreda em vez de causar dano)
}
```

#### Armadura
```typescript
interface ItemArmadura extends ItemBase {
  categoria: "armadura";
  subcategoria: "leve" | "pesada";
  bonus_defesa: number;        // ex: +2, +5
  penalidade_armadura: number; // ex: 0, -1, -3 (sempre <= 0)
  especial?: string;           // ex: armadura acolchoada fornece +2 Fortitude; armadura completa precisa ser sob medida
}
```

#### Escudo
```typescript
interface ItemEscudo extends ItemBase {
  categoria: "escudo";
  subcategoria: "leve" | "pesado";
  bonus_defesa: number;
  penalidade_armadura: number;
  dano_ataque: string;         // ex: "1d4 impacto" (leve), "1d6 impacto" (pesado)
}
```

#### Item de Aventura / Ferramenta / Vestuário
```typescript
interface ItemGeral extends ItemBase {
  categoria: "item-aventura" | "ferramenta" | "vestuario";
  beneficio?: string;          // bônus mecânico, ex: "+5 em testes de Ladinagem para ocultá-la"
  pericia_relacionada?: string;// ex: "Ladinagem", "Cura"
  precisa_empunhar: boolean;
  precisa_vestir: boolean;
}
```

#### Esotérico
```typescript
interface ItemEsoterico extends ItemBase {
  categoria: "esoterico";
  tipo_esoterico?: "varinha" | "cajado" | "orbe" | "cetro" | "medalhao" | "bolsa" | "luva" | "tomo" | "dedo";
  beneficio_magia: string;     // o efeito mecânico na magia
  instrumento_musical: boolean;// alguns esotéricos também são instrumentos (flauta mística, alaúde)
}
```

#### Alquímico Preparado

## 6. Observações de extração (armadilhas do pdftotext)

1. **Texto em duas colunas**: O `pdftotext -layout` embaralha quando o rodapé de página aparece no meio das colunas. A regra confirmada (PROGRESSO.md): **confirmar ordem pela imagem sempre** que uma linha de tabela parecer deslocada.

2. **Tabela 3-3 foi cortada em duas páginas** (impressas 143 e 144) com cabeçalho repetido "Tabela 3-3: Armas (Continuação)". No texto extraído isso virou duas seções separadas — o extrator precisa reconhecer a continuação como parte da mesma tabela.

3. **Dano duplo (barra `/`)**: Espada bastarda (`1d10/1d12`), Katana (`1d8/1d10`), Corrente de espinhos (`2d4/2d4`) têm barra separando dano de uma mão / duas mãos. O parser precisa tratar esse formato como dois valores.

4. **Crítico composto**: Pistola e Mosquete têm crítico `19/x3` (margem 19-20 + multiplicador x3). O campo `critico` precisa suportar ambos.

5. **Espaços "meio"**: Itens alquímicos, poções, pergaminhos aparecem com espaço `0,5` na tabela. O schema usa `"meio"` ou `0.5` (number).

6. **Itens sem preço (`—`)**: Clava, Bordão, Tacape, Funda têm preço zero (fabricáveis facilmente). Guardar como `null` ou `"—"` e documentar.

7. **Instrumentos musicais** aparecem tanto na subcategoria **Ferramentas** (alaúde élfico, flauta mística, instrumento musical, tambor das profundezas) quanto podem ser usados como **esotéricos** por bardos. Alguns itens têm dupla classificação.

8. **Sidebars/quadros coloridos** com regras importantes (Carga: Bastidores, Ataques Desarmados & Armas Naturais, Armas Improvisadas, Instrumentos Musicais, Regras de Venenos, Pratos Especiais, Fabricando Itens Superiores): o `pdftotext` captura o texto mas **sem a marcação visual de sidebar**. A imagem é necessária para confirmar que são quadros de regra (não corpo principal). Cada um deve virar uma entidade `regra`.

9. **Tabela 3-9 (Materiais Especiais)**: última tabela do capítulo, na impressa 167 — fim confirmado. Madeira Tollon **não serve para armaduras** (célula `—`), o que deve ser capturado como `null` no schema.
